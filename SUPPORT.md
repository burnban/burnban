# Support

## Community support

Use GitHub Issues for reproducible bugs and focused feature requests. Search
existing issues first and use the provided templates. Include `burnban version`,
operating system and architecture, install method, deployment mode, and minimal
sanitized reproduction steps.

Never post provider keys, `BURNBAN_TOKEN`, webhook URLs, raw prompts, request
bodies, databases, or agent log files. Replace account, agent, session, host,
and model identifiers when they are not necessary to reproduce the issue.

Questions about unsupported custom upstreams or local environments may be
answered on a best-effort basis. The project cannot troubleshoot provider
accounts, invoices, rate limits, or third-party agent behavior.

## Security and conduct

Do not report vulnerabilities in a public issue. Follow [SECURITY.md](SECURITY.md).
Conduct concerns are handled under [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Paid support boundary

Supporter and commercial tiers may receive prioritized response and onboarding,
but they do not gate fixes or features in this MIT repository. Security reports
are triaged by severity, not payment status. The open-source maintainers do not
promise a response time for ordinary community issues.
